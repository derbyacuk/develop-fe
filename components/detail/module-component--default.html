Multi Module Stage Combi

Updated version of the Module Stage Combi. Displays module information for a given course. Multiple pathways can be defined if required, and are automatically added to the module component. Increased speed, flexibility, and stability when compared to the single Module Stage Combi.

Enhancements

Increased flexibility

  • Updated JavaScript means that most properties are now expressed as variables.
  • Loading stages is now completed programatically, as is selecting stages. This allows for alternative triggers to be used.

    Increased stability

  • Much simpler, grid-based, markup increases compatability across browsers and reduces complexity and page weight
  • Simpler JavaScript delivers less error-prone code while increasing functionality and decreasing page weight

    Increased functionality

    • Multi Module Stage Combi adds the ability to create pathways within module data with built-in support for switching mechanisms.
    • Most attributes are now variable allowing for more flexible change/enhancement in the future

Technical Implementation

Markup

Requries module skeleton markup to be included in the page for module population. Please see the twig file for guidance on HTML - requried elements:

  • is-mobile-layout - used by JavaScript to detect if the view is currently mobile.
  • module-list-tabs - div for holding the stage selection tabs
  • module-list-container - main HTML structure for the module stage combi

Data

Module data should passed to the to the module component on instantiation.

Module data must follow the below structure:

{
    "components": [     // Array of 1 or more objects representing each pathway
        {
            "id": STRING,   // Unique ID representing the pathway "Art", "Dance", [...]
            "stages": [     // Array of one or more objects representing each stage
                {
                    "title": STRING,    // "Year 1", "Year 2", [...]
                    "categories": {
                        "title": STRING,
                        "modules": [ STRING, STRING, [...]] // An array of module IDs
                    }
                }
            ]
        }
    ],
    "modules": {    // An object containing one or more module objects
        STRING: {   // The property name must be a valid module ID
            "code": STRING      // The module ID, e.g: "4AZ502", "4AZ503", [...]
            "title": STRING     // The module title, e.g: "Expression and the Arts"
            "introduction": STRING  // Introductory text for the module (no markup)
            "details": STRING   // Detailed information text for the module (can use markup)
            "credits": STRING   // Number of credits expressed as a string, e.g: "20"
            "assessment": [STRING, STRING, STRING]    //  Array containing assessment method strings
        }
    }
}

Implementation

Firstly set up the data.

Include the javascript file or code.

Instantiate the module component, passing the module data to the constructor:

var moduleComponent = new ModuleComponent(moduleData);
moduleComponent.init();

Accessible methods

To load a different set of modules use

moduleComponent.createCourseModules("name-of-component")

Attach this to an event to trigger, for example, a pathway change, or a year change.
